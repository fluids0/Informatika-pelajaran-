🔄 Desimal → Biner (Metode Pembagian)

Bagi terus dengan 2, catat sisa pembagian dari bawah ke atas.

13 ÷ 2 = 6 sisa 1 6 ÷ 2 = 3 sisa 0 3 ÷ 2 = 1 sisa 1 1 ÷ 2 = 0 sisa 1 Baca dari bawah: 13₁₀ = 1101₂

🔄 Biner → Desimal (Nilai Posisi)

Kalikan setiap bit dengan 2 pangkat posisinya, lalu jumlahkan.

1 0 1 1₂ → Desimal? 1×2³ + 0×2² + 1×2¹ + 1×2⁰ = 8 + 0 + 2 + 1 = 11₁₀

🔄 Desimal → Heksadesimal

Bagi dengan 16, gunakan A–F untuk sisa 10–15.

254 ÷ 16 = 15 sisa 14 (E) 15 ÷ 16 = 0 sisa 15 (F) Baca dari bawah: 254₁₀ = FE₁₆

🔄 Biner ↔ Heksadesimal

Kelompokkan biner per 4 digit, setiap kelompok = 1 hex.

1010 1111₂ 1010 → A 1111 → F Hasil: AF₁₆
💡 Ingat! Komputer hanya mengenal 0 dan 1 (biner). Semua data — teks, gambar, video — disimpan dalam bentuk biner di dalam memori komputer.

Pengalamatan Memori

BAB 3

Setiap lokasi di memori komputer memiliki alamat unik, seperti nomor rumah di sebuah jalan. Alamat ini digunakan CPU untuk membaca atau menulis data.

🏠 Cara Kerja Pengalamatan

Setiap sel memori memiliki alamat yang dinyatakan dalam bilangan heksadesimal.

Alamat | Data ---------|------ 0x0000 | 01001000 (H) 0x0001 | 01100101 (e) 0x0002 | 01101100 (l) 0x0003 | 01101100 (l) 0x0004 | 01101111 (o)

Contoh: huruf "Hello" tersimpan di 5 alamat memori berurutan.

🔵 RAM (Random Access Memory)

Memori sementara — data hilang saat listrik dimatikan. Digunakan untuk menyimpan program yang sedang berjalan. Makin besar RAM, makin banyak program yang bisa dibuka bersamaan.

🟢 ROM (Read Only Memory)

Memori permanen — data tetap ada meski listrik mati. Berisi instruksi dasar (BIOS/UEFI) yang dijalankan saat komputer pertama kali dinyalakan. Tidak bisa diubah oleh pengguna biasa.

🟡 Cache Memory

Memori super cepat yang ada di dalam CPU. Menyimpan data yang paling sering diakses agar CPU tidak perlu bolak-balik ke RAM. Ukurannya kecil tapi kecepatannya sangat tinggi.

📍 IP Address juga adalah pengalamatan! Konsep pengalamatan tidak hanya di memori — di jaringan internet, setiap perangkat juga punya alamat IP yang unik agar data bisa dikirim ke tujuan yang tepat.

Jaringan Lokal dan Internet

BAB 4

🏠 LAN

Local Area Network

Jaringan dalam area kecil — rumah, sekolah, kantor. Contoh: komputer dan printer yang terhubung di ruang kelas.

🏙️ MAN

Metropolitan Area Network

Jaringan yang mencakup satu kota atau wilayah. Contoh: jaringan antar kampus atau gedung pemerintahan dalam satu kota.

🌍 WAN

Wide Area Network

Jaringan yang mencakup area sangat luas — antar kota, antar negara. Internet adalah WAN terbesar di dunia.

🔗 Link dan Node dalam Jaringan

📍 Node

Setiap perangkat yang terhubung dalam jaringan disebut node. Contoh node:

  • Komputer / laptop
  • Smartphone / tablet
  • Printer jaringan
  • Router dan switch
  • Server

📡 Link

Jalur komunikasi yang menghubungkan node-node. Bisa berupa:

  • Kabel UTP / LAN
  • Kabel fiber optik
  • Sinyal Wi-Fi (nirkabel)
  • Sinyal Bluetooth
  • Koneksi seluler (4G/5G)

🔀 Topologi Jaringan

Topologi Bus

Semua node terhubung ke satu kabel utama (backbone). Jika kabel utama rusak, seluruh jaringan lumpuh. Murah tapi kurang handal untuk jaringan besar.

  • Keuntungan: instalasi mudah dan murah
  • Kekurangan: rawan gangguan total
Topologi Star

Semua node terhubung ke satu perangkat pusat (switch/hub). Jika satu kabel rusak, hanya node itu yang terganggu. Paling umum digunakan di sekolah dan kantor.

  • Keuntungan: mudah troubleshoot, skalabel
  • Kekurangan: tergantung switch pusat
Topologi Ring

Data mengalir searah dalam bentuk lingkaran. Setiap node meneruskan data ke node berikutnya. Jika satu node rusak, data bisa terhenti.

Topologi Mesh

Setiap node terhubung langsung ke semua node lainnya. Sangat handal — jika satu jalur mati, ada jalur alternatif. Mahal karena butuh banyak kabel.

Topologi Tree

Gabungan beberapa topologi star yang dihubungkan secara hierarki. Cocok untuk jaringan besar seperti gedung bertingkat atau kampus.

📡 Komunikasi Data

Komponen Komunikasi Data

Pengirim (Sender)

Perangkat yang mengirimkan data — komputer, smartphone, sensor.

Media Transmisi

Jalur pengiriman data — kabel UTP, fiber optik, atau gelombang radio (Wi-Fi).

Penerima (Receiver)

Perangkat tujuan yang menerima dan memproses data.

Protokol

Aturan komunikasi yang disepakati — TCP/IP, HTTP, FTP.

Mode Komunikasi

Simplex — Data hanya mengalir satu arah. Contoh: siaran radio, remote TV.
Half-Duplex — Data bisa dua arah tapi bergantian. Contoh: walkie-talkie, radio HT.
Full-Duplex — Data bisa dua arah bersamaan. Contoh: telepon, video call, internet.

Terhubung ke Internet dengan Aman

BAB 5

✅ Tips Aman di Internet

  • Gunakan password yang kuat dan berbeda di setiap akun
  • Aktifkan verifikasi dua langkah (2FA)
  • Hanya buka website yang menggunakan HTTPS (gembok 🔒)
  • Jangan sambungkan ke Wi-Fi publik tanpa VPN
  • Update perangkat dan aplikasi secara rutin
  • Jangan klik tautan mencurigakan dari email/WA
  • Log out dari akun setelah selesai di komputer umum

⚠️ Ancaman di Internet

Phishing — Email/pesan palsu yang mengelabui kamu untuk memberikan data pribadi atau password.
Malware — Program jahat yang bisa mencuri data atau merusak perangkat. Termasuk virus, trojan, spyware.
Hoaks — Informasi palsu yang sengaja disebarkan untuk menyesatkan.
Penipuan Online — Tawaran palsu, belanja online palsu, investasi bodong.

🛡️ HTTP vs HTTPS

HTTP

Data dikirim tanpa enkripsi. Bisa dibaca siapa saja yang "mendengarkan" jaringan. Jangan masukkan data pribadi di situs HTTP!

http://contoh.com/login
HTTPS

Data dienkripsi menggunakan SSL/TLS. Hanya pengirim dan penerima yang bisa membaca data. Aman untuk transaksi dan login.

https://contoh.com/login 🔒

Media Sosial

BAB 6

✅ Manfaat Positif

  • Menghubungkan dengan teman dan keluarga
  • Mendapatkan informasi dan berita terkini
  • Media belajar dan berbagi pengetahuan
  • Membangun portofolio dan karier
  • Platform bisnis dan wirausaha

⚠️ Risiko Negatif

  • Kecanduan dan lupa waktu
  • Cyberbullying dan perundungan
  • Penyebaran hoaks dan disinformasi
  • Gangguan privasi dan data bocor
  • FOMO (Fear of Missing Out)

🌱 Etika Bersosmed

  • Pikirkan sebelum posting
  • Jangan sebarkan informasi belum terverifikasi
  • Hormati privasi orang lain
  • Gunakan bahasa yang sopan
  • Laporkan konten yang melanggar aturan
🔐 Jejak Digital — Semua yang kamu posting di internet meninggalkan jejak digital permanen. Bahkan setelah dihapus, data bisa tersimpan di server atau sudah di-screenshot orang lain. Berpikirlah dua kali sebelum memposting!

Mengkaji Kritis Informasi

BAB 7

Di era digital, kita dibanjiri informasi setiap saat. Tidak semua informasi itu benar. Kemampuan literasi digital sangat penting untuk memilah mana yang fakta dan mana yang hoaks.

🧭 Metode SIFT — Cara Mengecek Informasi

⏸️
STOP

Berhenti sejenak sebelum menyebarkan. Jangan langsung percaya atau share.

🔎
INVESTIGATE

Cari tahu sumber aslinya. Siapa yang menulis? Situs apa ini?

🔗
FIND COVERAGE

Cari sumber lain yang memberitakan hal yang sama. Apakah berita itu ada di media terpercaya?

📜
TRACE CLAIMS

Lacak klaim ke sumber aslinya. Sering kali informasi sudah diputarbalikkan dari aslinya.

📰 Ciri-Ciri Hoaks

  • Judul sensasional / clickbait berlebihan
  • Tidak ada nama penulis atau sumber jelas
  • URL mencurigakan atau domain aneh
  • Foto/video diambil dari konteks berbeda
  • Menggunakan kata "VIRAL", "SEGERA SHARE"
  • Tanggal tidak ada atau sudah lama
  • Isi bertentangan dengan logika umum

✅ Sumber Informasi Terpercaya

  • Media berita ber-dewan redaksi dan terverifikasi Dewan Pers
  • Website pemerintah (.go.id, .gov)
  • Lembaga ilmiah dan akademik (.ac.id, .edu)
  • Situs fact-checking: Cek Fakta, TurnBackHoax
  • WHO, Kemenkes untuk info kesehatan
  • BMKG untuk info cuaca dan bencana

Cyberbullying & Keamanan Digital

BAB 8
Cyberbullying adalah perilaku intimidasi, pelecehan, atau perundungan yang dilakukan melalui media digital seperti internet, media sosial, atau pesan teks. Sama berbahayanya dengan perundungan di dunia nyata, bahkan bisa lebih parah karena berlangsung 24 jam dan meninggalkan jejak.

Jenis-Jenis Cyberbullying

😡 Flaming

Mengirimkan pesan kasar, penuh amarah, dan kata-kata menyakitkan kepada seseorang secara online. Sering terjadi di kolom komentar atau grup chat.

👣 Harassment (Pelecehan)

Mengirimkan pesan kasar, menghina, atau mengancam secara berulang-ulang kepada seseorang. Bisa melalui DM, komentar, atau email.

🕵️ Cyberstalking

Menguntit seseorang secara online — memantau semua aktivitas, lokasi, dan kehidupan korban tanpa izin, seringkali disertai ancaman.

📢 Outing (Membocorkan Rahasia)

Menyebarkan informasi pribadi, rahasia, atau foto memalukan seseorang tanpa izin ke khalayak umum.

🎭 Impersonation (Penyamaran)

Berpura-pura menjadi orang lain secara online untuk merusak reputasinya, menipu teman-temannya, atau melakukan hal memalukan atas nama korban.

🚷 Exclusion (Pengucilan)

Sengaja mengucilkan, tidak mengajak, atau mengeluarkan seseorang dari grup online dengan tujuan menyakiti perasaannya.

🎣 Trickery (Penipuan)

Menipu seseorang agar mau membocorkan rahasia atau informasi pribadi, lalu menyebarkannya ke orang lain.

🌊 Doxing

Menyebarkan informasi identitas pribadi seseorang (nama lengkap, alamat, nomor HP, dll.) secara online tanpa izin, biasanya dengan tujuan mempermalukan atau membahayakan.

🆘 Jika Kamu Korban

  • Jangan balas pesan pelaku
  • Simpan bukti (screenshot)
  • Blokir pelaku
  • Ceritakan ke orang tua/guru yang dipercaya
  • Laporkan ke platform media sosial
  • Hubungi hotline Kominfo: 159

👀 Jika Kamu Saksi

  • Jangan ikut menyebarkan atau menyukai konten bullying
  • Dukung korban secara pribadi
  • Laporkan konten ke platform
  • Beritahu guru atau orang dewasa
  • Jadilah teman bagi korban

🌟 Pencegahan

  • Berpikir sebelum mengirim pesan
  • Jaga privasi akun media sosial
  • Hati-hati berteman dengan orang tidak dikenal
  • Edukasi diri tentang etika digital
  • Diskusikan dengan keluarga

🔑 Membuat Kata Sandi yang Kuat

✅ Syarat Password Kuat

  • Minimal 12 karakter (makin panjang makin baik)
  • Kombinasi huruf BESAR dan kecil
  • Mengandung angka (0–9)
  • Mengandung simbol (!@#$%^&*)
  • Tidak menggunakan nama, tanggal lahir, atau kata umum
  • Berbeda untuk setiap akun

Gunakan metode passphrase — ambil kata-kata acak dan gabungkan:

Contoh kalimat: "Kucing saya makan nasi 3 kali!" Password: KuS@y@M@k4nN4s1#3k! Atau pakai frasa: "Aku lahir di Bandung tahun 2009" → Al@dB@ndung#2009!

Jaringan Seluler

BAB 9

Generasi Jaringan Seluler

1G — Generasi Pertama (1980-an)

Analog, hanya untuk suara (telepon). Kualitas buruk, mudah disadap, tidak ada enkripsi data.

2G — GSM/CDMA (1990-an)

Digital, suara lebih jernih + SMS. Kecepatan data sangat lambat (9,6–114 Kbps). Generasi pertama yang bisa mengirim pesan teks.

3G — UMTS/WCDMA (2000-an)

Internet mobile mulai nyaman. Kecepatan 384 Kbps–7,2 Mbps. Sudah bisa browsing, email, video call standar.

4G LTE — Long Term Evolution (2010-an)

Era smartphone modern. Kecepatan 50–300 Mbps. Streaming video HD, media sosial, gaming online sudah lancar.

5G — Generasi Kelima (2020-an)

Ultra cepat dan latensi sangat rendah. Kecepatan bisa mencapai 1–10 Gbps. Mendukung IoT, kendaraan otonom, dan smart city.

📶 Faktor yang Mempengaruhi Kekuatan Sinyal

📍 Jarak dari BTS

Semakin jauh dari menara BTS (Base Transceiver Station), sinyalnya semakin lemah. Sinyal melemah mengikuti hukum kuadrat terbalik.

🏢 Penghalang Fisik

Tembok tebal, gedung beton, pegunungan, dan bangunan tinggi dapat memblokir atau memantulkan sinyal, menyebabkan zona mati (dead zone).

📡 Interferensi

Sinyal dari perangkat lain pada frekuensi yang sama dapat saling mengganggu. Semakin banyak pengguna di satu sel, semakin terbagi bandwidth-nya.

🌦️ Kondisi Cuaca

Hujan deras, badai petir, dan kelembapan tinggi dapat menyerap dan mengganggu gelombang radio, terutama pada frekuensi tinggi (5G mmWave).

📶 Frekuensi yang Digunakan

Frekuensi rendah (700 MHz) jangkauannya luas tapi kecepatan lebih rendah. Frekuensi tinggi (mmWave) sangat cepat tapi jangkauan pendek dan mudah terhalang.

👥 Kepadatan Pengguna

Di tempat ramai seperti konser atau stadion, banyak orang terhubung ke BTS yang sama sehingga koneksi melambat (network congestion).

Routing pada Jaringan Komputer

BAB 10

Routing adalah proses memilih jalur terbaik untuk mengirimkan paket data dari sumber ke tujuan melalui jaringan. Perangkat yang melakukan routing disebut router.

🔵 Static Routing

Jalur ditentukan secara manual oleh administrator jaringan. Tidak berubah kecuali diubah secara manual.

Keuntungan
  • Stabil dan dapat diprediksi
  • Tidak butuh resource CPU tinggi
  • Cocok untuk jaringan kecil
Kekurangan
  • Tidak fleksibel jika topologi berubah
  • Perlu konfigurasi manual tiap router

🟢 Dynamic Routing

Router secara otomatis mempelajari dan memperbarui tabel routing menggunakan protokol routing.

Keuntungan
  • Otomatis beradaptasi jika ada perubahan
  • Cocok untuk jaringan besar dan kompleks
Kekurangan
  • Menggunakan bandwidth untuk update routing
  • Lebih kompleks dikonfigurasi

⚙️ Perangkat Jaringan

🔀 Router

Menghubungkan dua jaringan berbeda (misal LAN ke internet). Bertugas memilih jalur terbaik untuk paket data. Setiap rumah/kantor punya router untuk akses internet.

🔁 Switch

Menghubungkan perangkat-perangkat dalam satu jaringan lokal (LAN). Lebih cerdas dari hub — mengirim data hanya ke perangkat tujuan, bukan ke semua port.

📡 Access Point (AP)

Memancarkan sinyal Wi-Fi agar perangkat nirkabel bisa terhubung ke jaringan kabel. Biasanya sudah terintegrasi di dalam router Wi-Fi modern.

🖧 NIC (Network Interface Card)

Kartu jaringan yang terpasang di komputer/laptop sebagai antarmuka untuk terhubung ke jaringan. Memiliki alamat fisik unik yang disebut MAC Address.

🛠️ Konfigurasi IP di Windows

# Cara 1: Lihat konfigurasi jaringan saat ini ipconfig # Cara 2: Detail lengkap termasuk MAC Address dan DNS ipconfig /all # Cara 3: Uji koneksi ke perangkat lain (ping) ping 192.168.1.1 ping google.com # Cara 4: Lacak rute paket data tracert google.com

📋 Konfigurasi IP Manual (Static)

Untuk mengatur IP statis di Windows: Control Panel → Network → Adapter Properties → TCP/IPv4

IP Address : 192.168.1.10 Subnet Mask : 255.255.255.0 Gateway : 192.168.1.1 DNS Server : 8.8.8.8 (Google) atau 1.1.1.1 (Cloudflare)

Penjelasan setiap komponen:

IP Address — Identitas unik perangkat di jaringan lokal.
Subnet Mask — Menentukan batas jaringan. /24 = 255.255.255.0 artinya 254 host tersedia.
Default Gateway — Alamat router — pintu keluar ke jaringan lain / internet.
DNS Server — Menerjemahkan nama domain (google.com) menjadi IP address.

🔄 Cara Kerja DHCP

DHCP (Dynamic Host Configuration Protocol) otomatis memberikan IP address ke perangkat yang baru bergabung ke jaringan. Prosesnya 4 langkah:

DHCP Discover

Perangkat baru menyiarkan pesan ke seluruh jaringan: "Ada DHCP server di sini?"

DHCP Offer

DHCP server (biasanya router) merespons: "Ada! Saya tawarkan IP 192.168.1.50 untukmu."

DHCP Request

Perangkat menerima tawaran: "Saya minta IP 192.168.1.50 tersebut."

DHCP Acknowledge

Server mengkonfirmasi: "Disetujui! IP 192.168.1.50 adalah milikmu selama 24 jam (lease time)."

🧩 DNS — Domain Name System: Saat kamu ketik www.google.com di browser, komputer tidak langsung mengerti. DNS bertugas menerjemahkannya menjadi IP Address yang bisa dipahami komputer, misalnya 142.250.185.46. DNS ibarat buku telepon internet.
